# About `fabric_workspace` folder

This is an **autogenerated folder** that synchronizes with the Microsoft Fabric workspace and enables automated deployment.

## Purpose

This folder follows the [Fabric workspace Git format](https://learn.microsoft.com/fabric/cicd/git-integration/git-get-started) and serves as:

- **Source of truth** for Fabric workspace items (notebooks, lakehouses, data agents, ontologies)
- **Deployment package** for CI/CD via [fabric-launcher](https://github.com/microsoft/fabric-launcher)
- **Version control** integration using Fabric's Git synchronization

## Deployment

This workspace is deployed using:

- **[fabric-launcher](https://github.com/microsoft/fabric-launcher)** — Python package for programmatic Fabric deployments
- **[fabric-cicd](https://pypi.org/project/fabric-cicd/)** — Parameterization and environment-specific configuration via `parameter.yml`
- **Fabric Git Integration** — Native sync between this folder and the Fabric workspace

## Structure

Each item is a folder named `{name}.{Type}/` containing:
- `.platform` — Fabric metadata file (managed automatically)
- Type-specific content files (e.g., `notebook-content.py`, `lakehouse.metadata.json`)

See [`.github/instructions/fabric-workspace.instructions.md`](../../../.github/instructions/fabric-workspace.instructions.md) for detailed conventions.
